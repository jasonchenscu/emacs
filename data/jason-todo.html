2 Steps to add one chart in sustaining metrics

Login to the system as your user account (cheny7)

  • Update on client side
    • Modify pods.xml in dir /opt/lampp/htdocs/metrics/data
    • Add new pod panel in the bottom with view 'sustaining', modify the data source to the proper REST link of server side


  • Update on server side
    • Modify sustainingcontroller.rb in dir /opt/sustainingmetrics/app/controllers
    • Add new action in the controller (e.g. assignment)


3 Requirement draft after discuss with Rulian

  1. Metrics of most interest to upper mgmt:

61 Total # of backlog bugs by week; (ideally, we want to be able to drill down into the bugs by component; by severity; by release found in, etc.);
61 # of backlog bugs added by week;
61 # of sustaining bus resolved per week


  1. Metrics that helps driving the bugzilla scrubbing effort:

61 Total # of backlog bugs by week;
61 Total # of backlog bugs tagged with sustaining keyword by week;
61 Total # of backlog bugs reviewed by week;


  1. Metrics that shows sustaining project team member status:

61 Bug assignment status by sustaining team member (can drill down into status by geography location)
61 Bug resolved status by sustaining team member (can drill down into status by geography location)


  1. Metrics that shows sustaining focus by component:

61 Bug assignment status by component
61 Bug resolved status by component


  1. Metrics that shows incoming bugs trend, by component

61 All new open bugs with GA version by component
